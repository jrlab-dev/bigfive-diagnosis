### 3-2. ② 診断テスト（test.html）

**目的：** 30問の回答でタイプコードを決定

**フロー：**
```
スタート画面
  → 説明文（「30問・3分で完了」）
  → ニックネーム入力（任意）
  → 質問1〜30（1問ずつ表示 or 全問一括表示）
  → 計算処理（JavaScript）
  → result.htmlにリダイレクト
```

**UI仕様：**
- プログレスバー表示（何問目/30問）
- 回答は1〜5の5段階ボタン（タップしやすい大きさ）
- 前の質問に戻れる「戻る」ボタン
- 途中保存なし（全問必須）

**スコア計算：** 07ファイルの計算式をJavaScriptで実装
- 30問の回答を5因子スコアに変換
- 各因子を1〜5段階に変換
- `type=XXXXX` のURLパラメータでresult.htmlへ

**URL例：** `/bigfive/result.html?type=34251`

**コンテンツ構成：**
```

  ■■□□□ 外向性 2/5
  ■■■■■ 協調性 5/5
  ■□□□□ 安定性 5/5（N逆）

[タイプ説明文]
  〜400字の説明文（09ファイルのテンプレートから生成）

[強み・弱み]
  強み：[リスト]
  弱み：[リスト]

[恋愛スタイル]
  タイプ名 ＋ 100字説明

[向いている職業]
  職業リスト（3〜5個）

[相性の良いタイプ TOP5]
  ※08ファイルの計算式で上位5タイプを表示
  各タイプへのリンク付き

[SNSシェアボタン]
  X（Twitter）シェア ＋ コピーボタン

[再診断・相性チェック CTAボタン]
```

**SEO対策：**
```html
<title>ビッグファイブ「落ち着いた調停者」(#34251) - 性格診断3125タイプ</title>
<meta name="description" content="ビッグファイブ性格タイプ「落ち着いた調停者」の詳細解説。開放性3・誠実性4・外向性2・協調性5・安定性5の特徴、強み・弱み、相性の良いタイプを解説。">
```

### 3-5. ⑤ タイプ詳細ページ（types/XXXXX.html）

**目的：** 3125タイプそれぞれの静的ページ → **SEOの核心**

**実装方式：**

3125ページを手動作成は不可能なため、**JavaScriptで動的生成**する方式を採用：

```
types/index.html?code=34251
```

または、**静的ページ生成スクリプト（Node.js）** で3125個のHTMLを事前生成：
```
node generate_type_pages.js  // → types/34251.html を3125個生成
```

**SEO的には静的ページ生成が有利**（Google がインデックスしやすい）

**各ページのコンテンツ：**
- result.htmlと同等の内容
- + 「このタイプの有名人（仮）」
- + 「このタイプのあるある行動」5つ

## 5. 技術仕様

### 5-1. 技術スタック

| 項目 | 技術 | 理由 |
|------|------|------|
| フロントエンド | HTML/CSS/JavaScript（Vanilla） | GitHub Pages対応・シンプル |
| CSSフレームワーク | Tailwind CSS（CDN） or カスタムCSS | |
| データ格納 | JavaScript定数（インメモリ） | サーバー不要 |
| 3125ページ生成 | Node.js スクリプト（事前生成） | SEO対応 |
| 分析 | Google Analytics 4 | 無料 |
| ホスティング | GitHub Pages | 無料・既存環境 |

### 5-2. データファイル構成

```
bigfive/
├── js/
│   ├── type-names.js      // 3125タイプ名生成ロジック
│   ├── type-desc.js       // 説明文フラグメント
│   ├── compat-calc.js     // 相性スコア計算
│   ├── test-logic.js      // 診断テストロジック
│   └── result-render.js   // 結果ページ描画
├── css/
│   └── bigfive.css        // スタイル
└── data/
    └── fragments.js       // 説明文フラグメントデータ
```

### 5-3. パフォーマンス設計

- 全ロジックはJavaScript（クライアントサイド）→ サーバーゼロ
- 3125タイプ分のデータは軽量（各タイプ〜500バイト程度 × 3125 ≈ 1.5MB）
- 静的ページは必要な時だけ生成（Node.jsスクリプト）
